The idea in one line

A specialist for every marketing job.

Skills are markdown files that hand an AI agent the right framework, checklist, and best practices the moment you start a marketing task. Instead of one blank prompt, you get the conversion playbook for a pricing page, the deliverability rules for a cold-email sequence, or the schema spec for an SEO fix — applied automatically.

◆

One shared foundation

product-marketing captures your positioning and audience once. Every other skill reads it first, so advice fits your product — not a generic template.

⚙

Triggered, not searched

Each skill lists the phrases that summon it. Say "this landing page isn't converting" and the CRO skill loads itself — no command to remember.

∞

They build on each other

copywriting ↔ cro ↔ ab-testing, seo-audit ↔ schema ↔ ai-seo. Skills cross-reference so a task pulls in everything it needs.

Two libraries, no confusion

Where this overlaps gstack — and which to use

A few skills exist in both this library and gstack's marketing layer. They coexist on purpose; here's the rule for each so you never wonder which one fired.

name collision
copywriting·cro·content-strategy·customer-researchusemarketingskills

These four names exist in both the marketingskills library and gstack's marketing layer. On disk they resolve to marketingskills (the v2 versions), and trigger-routing prefers them too — the v2 descriptions pack 10–20 trigger phrases. Net: you always get the marketingskills version here.

same domain, different job
socialvssocial-media (gstack)useboth — on purpose

social here is content-creation focused (what to post, how to write it). gstack's social-media is growth-mechanics focused (platform-by-platform follower playbook). Use the one that matches the task.

granular vs broad
seo-audit · ai-seo · schema · …vsseo (gstack)usetask-dependent

marketingskills splits SEO into focused skills (seo-audit, ai-seo, programmatic-seo, schema, site-architecture). gstack's seo is one broad-strategy skill. Reach for a granular one for a specific task, gstack's for whole-funnel strategy.

granular vs whole-journey
emails · cold-email · churn-prevention · onboardingvslifecycle (gstack)usetask-dependent

These cover specific lifecycle phases; gstack's lifecycle is one whole-journey email playbook. Specific phase → use the focused skill; whole program → gstack's lifecycle.

near-duplicate
pricingvspricing-strategy (gstack)usepricing

Overlapping domain, different names. Default to marketingskills pricing (packaging + monetization); gstack's pricing-strategy is the v1 analog.

Two minutes

Install

Drop the whole library into any agent, or cherry-pick the skills you want. Files land in .agents/skills/ and symlink into .claude/skills/ for Claude Code.

terminal — npx skills
# install everything
npx skills add coreyhaines31/marketingskills

# or cherry-pick
npx skills add coreyhaines31/marketingskills --skill cro copywriting ads

# list what's available
npx skills add coreyhaines31/marketingskills --list

# Claude Code plugin install (alternative)
/plugin marketplace add coreyhaines31/marketingskills → /plugin install marketing-skills
